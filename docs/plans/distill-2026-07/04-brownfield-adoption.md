## 1. Intention

The brownfield track exists to make an existing codebase converge to the same canonical doc set and operational layer a greenfield project gets — by reading code instead of interviewing from scratch, and interviewing the user only for what code cannot reveal. Its five skills split one intention: **scan reads the code once; the extracts translate mechanism into meaning; infra-adopt bolts on tooling and turns accumulated debt into a steerable roadmap.** The section's distinctive stances — infer-first-interview-last, mint-nothing-without-rationale, empty-ledger-is-honest, additive-never-destructive — are the parts that must survive any cut intact.

Per-file intention map:

- **groundwork-scan/instructions.md** — cause a session to build a deterministic structural map, read the repo partition-by-partition at a user-confirmed depth without overflowing context (write-and-purge, resumable), and leave concern-split findings files that the three extracts consume without re-reading code.
- **groundwork-scan/references/digest-schema.md** — pin the one bounded digest shape both fan-out paths produce, and route each field to the findings file its downstream consumer reads, so no signal (especially interface surfaces and missing contracts) is dropped between scan and extract.
- **groundwork-scan/references/exclusions.md** — keep every read pointed at meaning-bearing files: never burn context on vendored/generated noise, always read contract-bearing files first, at every depth.
- **groundwork-scan/templates/** — give the scan a resumable state shape and give each extract a predictable header set to find its signals under.
- **groundwork-product-brief-extract** — cause the agent to recover the product vision code embodies (not describe the code), propose it to the user, interview only the why/who/success gaps, and commit a brief indistinguishable from greenfield's — or bring an existing/foreign brief forward without overwriting it.
- **groundwork-design-system-extract** — recover the design language already encoded in config and components as concrete values with semantic roles, confirm the interface-type set per surface, interview only for intent (deliberate vs accreted), and commit `docs/design-system.md` + `brand-tokens.json` so infra-adopt can brand the `./dev` CLI.
- **groundwork-architecture-extract** — reconstruct the architecture from exact facts (findings + repo-map), confirm rather than invent, mint ADRs only where the user supplies rationale, and commit the full architecture artifact set (index, domain stubs, ADRs, surface registry + machine twin, capability-ports, drift frontmatter) plus the heaviest gap-ledger contribution.
- **groundwork-infra-adopt** — adopt existing services into docs and bolt on the operational layer without touching application code (never run a service generator; merge, never clobber compose), verify the stack boots, convert the gap ledger into `docs/maturity.md` as prioritised schedulable work, and hand a completed setup to the bet loop through the orchestrator.

## 3. Distillation moves

**M1 — Root-fix the revise cap (BROWN-1).** Move the "hard stop + fresh contract↔body desyncs mean an unreconciled Downstream Context file" diagnostic into operating-contract Protocol 8 (one sentence each), then reduce all four skills' revise-loop steps to the same one-liner: "On REVISE, apply all 🔴 findings and re-review; Protocol 8's revise cap and hard-stop rule apply." Delivery improves because the diagnostic then reaches *every* Protocol 8 caller (today product-brief-extract and infra-adopt lack it), and future skills inherit it. Requires FLAG(1) coordination — the contract is not this section's file.

**M2 — Single-source the exclusions (BROWN-2).** `templates/scan-state.json` stops carrying a literal `exclusions` array: replace with `"exclusions": "see .groundwork/skills/groundwork-scan/references/exclusions.md"` or have Stage 1 populate the array *from* exclusions.md when writing the classification (the instruction already says "load it now; it governs every read"). Stage 3a's sub-agent brief then passes the reference path, not a stale array. Fixes an active drift and makes exclusions.md the one writer.

Keep infra-adopt step 9's "route through the orchestrator — do not load `groundwork-bet` directly" (a real observed failure mode) but cut its mechanism re-explanation to one clause.

**M4 — State the frontmatter exemption once (BROWN-4).** Keep the rule in both skills ("Stamp no drift frontmatter — the brief/design system is exempt by design") but carry the *rationale and the four-path list* only in product-brief-extract (first occurrence in the track); design-system-extract cites "same exemption as the brief (see `groundwork-check`'s code-coupled scope)".
