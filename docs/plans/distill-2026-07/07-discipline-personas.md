## 1. Intention

**The section's reason to exist.** GroundWork's setup and bet workflows choreograph *conversations*; they do not carry *expertise*. The personas are where the expertise lives: a stance (senior architect / senior product leader), a small always-loaded judgment layer (principles, first checks, safety gates, handoffs), and a decision-time reference library loaded one shape at a time. The intended session outcome: when an adopting workflow (`groundwork-architecture`, `groundwork-product-brief`, bet `01-discovery`/`02-design`/`05-validation`) reaches a structural or product decision, the agent (a) recognizes which discipline owns it via the risk split (product = value+viability, designer = usability, architect+engineers = feasibility), (b) loads exactly one or two references matching the decision shape, (c) advises with trade-offs and named costs rather than menus, (d) refuses to silently re-open committed decisions, and (e) hands implementation off. The pattern is deliberately *persona-in-a-workflow-route*: the orchestrator never routes to a persona directly (`groundwork-orchestrator/SKILL.md` line ~147 states this), so the SKILL.md must work as an adopted overlay, not a phase.

**Per-file intention map.**

- **`groundwork-architect/SKILL.md`** — install the architect's identity and judgment layer in ~1,500 words: 7 carried principles (a mirror of the corpus manifesto), 5 operating rules, 5 pre-advice fact checks, a 17-row decision-shape → reference routing table, handoff boundaries, 7 safety gates (the conversation-time catch-list), and output quality bars ("boundaries are not an org chart… contracts are not endpoint lists"). It exists so the *always-loaded* cost of the persona stays small while 12.6k words of doctrine stay one hop away.
- **`groundwork-product/SKILL.md`** — the exact structural sibling for product: problem-before-solution, riskiest-assumption-first, appetite-not-estimate, falsifiable success signals, 7-row routing table, 6 safety gates.
- **Architect references (17 files)** — each is meant to be a *decision-time tool*: intro stating why the decision class is expensive to get wrong, numbered design decisions, a distinctive judgment call (a table or fork), and "Antipatterns to catch" as the conversational tripwires. Grouped by intention:
  - *Structure*: `core-and-boundaries` (the highest-leverage file: core/surface model, when a boundary is justified, dependencies-inward, boundary = test seam), `surface-architecture` (owns the seam, not the pixels).
  - *Contracts & communication*: `api-and-contracts`, `integration-patterns`, `realtime-and-async`, `durable-execution`, `data-architecture` — make the guarantee, not the convenience, pick the pattern.
  - *Operational posture*: `reliability`, `performance-and-scale`, `observability`, `platform-and-delivery` — design-time budgets/SLOs/instrumentation so "later" never happens.
  - *Trust*: `security-and-trust`, `identity-and-access` — trust model decided with the boundaries.
  - *AI*: `ai-native-architecture` (interfaces agents consume + model-in-the-loop features), `agentic-systems` (agent as actor).
  - *Governance*: `decision-records` (the ADR convention the whole framework leans on — Protocol 10, reversals, bet validation Step 7), `evolutionary-architecture` (fitness functions, strangler fig, advice process).
- **Product references (7 files)** — same tool shape: `discovery-and-opportunity` (opportunity-solution tree, evidence weighting), `product-risks` (the four risks + ownership + riskiest-first), `requirements-and-specs` (jobs, journeys, stable-ID FRs, Given/When/Then), `success-metrics-and-signals` (North Star, counter-metrics, falsifiability), `shaping-and-appetite` (per-item: outcome/appetite/stakes), `scope-and-sequencing` (across-work: bets, opportunity cost, no-gos), `ai-native-product` (outcome envelope, evals as product work, three cost layers).
- **`sync-anchor.md` (both)** — pin the SHA-256 of every corpus source a persona distils so a corpus edit forces a same-commit persona review (`scripts/check_sync_anchors.py`, run by `./dev test contracts`). The intention is drift-proof duplication: references may restate the corpus *because* the anchor makes silent divergence impossible.

**Overall verdict up front:** this is one of the healthiest sections in the repo — recently built (2026-06), tight prose, consistent spines across all three personas, clean adoption seams that point rather than restate. The distillation opportunity is not bulk removal; it is (1) consolidating the second-wave architect references that mirrored source files 1:1 instead of decision shapes, (2) removing doctrine that is stated in three places and will drift, (3) de-dating claims that will read stale in shipped projects, and (4) making the sync-anchor point at the reference it protects.
