# Archived plans

Completed plans live here. A plan moves into `archive/` once its workstreams are
delivered into the working tree (live-simulation/verification debt that is human-gated
and tracked elsewhere does not block archival). They are kept — not deleted — because a
plan is the design rationale for the shape a subsystem now has: read the plan that shaped
a subsystem before reworking it. The `CHANGELOG.md` links into these for the per-slice
record.

Active plans (with real unbuilt work remaining) stay one level up in `docs/plans/`.
