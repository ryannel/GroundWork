# Team Handbook

Stockroom plans and ships through the work-unit system defined in
`workdocs/how-we-work.md` — read it before touching anything.

- Shaping or delivering a work unit → hand off to the `workunit-lead` skill.
- Service code lives in `services/`; tests in `tests/`.
- Never author planning docs outside `workdocs/`, and never edit
  `.agents/skills/` directly — it is generated by `./dev sync skills`.
